# Repo Map

- Start medium and large tasks with a task brief: `Goal`, `Context`, `Constraints`, `Done when`.
- For medium tasks, write a plan before editing code.
- For large or ambiguous work, create or update a live exec plan in `docs/exec-plans/active/`.
- Treat this file as a map only. Keep durable facts in `docs/` and reusable workflows in `.agents/skills/`.
- Read relevant repo docs before coding.
- Never finish with code only. Report verification and remaining risk.
